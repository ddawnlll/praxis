# PRAXIS v0.2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use compose:subagent (recommended) or compose:execute to implement this plan task-by-task.

**Goal:** Build PRAXIS v0.2 — server runtime, Circuit Breaker, schema fixes, ACCP reports, TestOutputParser, infrastructure

**Architecture:** Six independent workstreams that can execute in parallel. The server package (`packages/server/`) adds Hono HTTP + SSE streaming. Circuit Breaker adds kernel safety. Schema patches fix the core schema. ACCP reports extend the report generator. TestOutputParser adds test framework parsing.

**Tech Stack:** TypeScript strict, Bun runtime, Hono (server), SSE (streaming), YAML (reports)

## Global Constraints

- All packages use TypeScript strict mode
- No copying from `pi/` forbidden list packages
- All new code must have tests
- Server binds to 127.0.0.1 only
- Server must not contain gate logic (thin dispatch layer)
- Circuit Breaker lives in kernel, not server
